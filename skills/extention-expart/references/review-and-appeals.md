# Review Process, Enforcement, and Appeals

## Review Basics
- **Timing**: Usually a few days, but can take several weeks.
- **Factors**: New developers, sensitive permissions (e.g., `<all_urls>`), and large codebases increase review time.
- **Cancel Review**: You can cancel a pending review up to 6 times per day if you find a bug before review finishes.

## Review Outcomes
- **Approved**: No violations found; item is published.
- **Rejected**: Submission fails review. Developer is notified of the violation.
- **Warning**: Published item found with minor violation. Typically given 7-30 days to fix before takedown.
- **Takedown**: Published item removed immediately for moderate/serious violations. 404 error on store page.
- **Account Suspension**: Egregious or repeated violations.

## Enforcement Types
- **Malware**: Immediate disable on user devices; no notification to developer; permanent suspension likely.
- **Deceptive Behavior**: Intentional evasion of review.
- **Repetitive Content**: Avoid publishing near-identical testing and production versions.

## Appeals & Support
- **Review Verdict Appeal**: Use the "Appeal" button on the item detail page in the dashboard.
- **Account Suspension Appeal**: Use the "Appeal" button on the account page.
- **One Stop Support**: Use for trader verification, transfer requests, or dashboard stats issues.
- **EU P2B Regulations**: European Union developers can raise concerns via the support form.

## Best Practices for Approval
- **Minimal Permissions**: Only request what is strictly necessary.
- **Code Readability**: Avoid obfuscation; provide author-authored code where possible.
- **Disclosures**: Be accurate and up-to-date in the "Privacy" tab.
